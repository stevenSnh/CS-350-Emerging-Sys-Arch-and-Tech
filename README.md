# CS-350-Emerging-Sys-Arch-and-Tech

This repository contains my project from CS 350 Emerging Systems Architecture and Technology. This project demonstrates my ability to write software that controls hardware components, use GPIO inputs and outputs, work with sensors, create state machines, and communicate with devices via interfaces such as I2C and UART. The smart thermostat project was designed to simulate a connected thermostat system. The program reads room temperature from an AHT20 sensor via I2C, displays information on a 16x2 LCD screen, and uses LEDs to indicate heating and cooling behavior. The user can press a button to cycle through Off, Heat, and Cool modes. One button decreases the set point, and the other increases it. The program uses a state machine to control the thermostat mode. It also sends the current state, temperature, and set point through UART every 30 seconds to simulate communication with a server. I did particularly well at connecting multiple hardware components into a working project. The final program combined button interrupts, PWM LED behavior, I2C temperature readings, LCD output, a state machine, and UART communication. I also created a state machine diagram to document how the thermostat changes between modes. I think the area where I could improve is testing more edge cases before combining all hardware components. I could create more automated tests for invalid input, sensor communication failures, and unusual temperature values. 

I think I could also improve the program by adding better error logging and more detailed comments for future developers. The main tools and resources I used during these projects included Python, Raspberry Pi, GPIO Zero, Visual Studio Code, Linux commands, I2C devices, UART communication, LCD displays, and state machine diagrams. I also used course resources, hardware documentation, and testing to troubleshoot. The most transferable skills from these projects are Python programming, debugging hardware and software together, GPIO control, sensor integration, serial communication, state machine design, and documenting system behavior. These skills can transfer to future work in embedded systems, automation, IoT, manufacturing systems, cybersecurity, and software development. I made the projects more maintainable by separating major tasks into functions, naming them to describe the hardware or action they control, and documenting the state transitions. The thermostat project is adaptable because the set point, GPIO pins, timing, and hardware behavior can be adjusted without rewriting the entire program. The state machine diagrams also help explain the design to someone reviewing the code later.
